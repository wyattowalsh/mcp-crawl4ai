## What is Crawl4AI MCP?

Crawl4AI MCP is a **high-performance server implementation** that connects the [Model Context Protocol (MCP)](https://modelcontextprotocol.ai) with advanced web crawling capabilities. It enables large language models like Claude to interact with the live web, retrieving real-time information and structured content from websites.

This server is designed to be:

- **Fast and efficient** - Optimized for quick response times with adaptive concurrency
- **Easy to integrate** - Compatible with any MCP client (Claude, Cursor, custom apps)
- **Reliable** - Robust error handling and recovery, with detailed logging
- **Flexible** - Multiple transport options, crawling configurations, and output formats
- **Developer-friendly** - Clear API, comprehensive documentation, and easy extensibility
- **Production-ready** - Scalable architecture suitable for individual and enterprise use

## Quick Start

Once installed, you can start the server with the `crawl4ai-mcp` command:

```bash
# Option 1: Default - Start with STDIO transport
# (Ideal for direct client integration like Claude Desktop)
crawl4ai-mcp serve

# Option 2: Start with HTTP/SSE transport on localhost port 8000
# (Requires [http] extras: pip install crawl4ai-mcp[http])
crawl4ai-mcp serve --transport http --host 127.0.0.1 --port 8000

# Option 3: Start with a configuration file
# (Overrides command-line args and defaults)
crawl4ai-mcp serve --config my_config.yaml

# Option 4: See all available options and help
crawl4ai-mcp serve --help
```

## Architecture

Crawl4AI-MCP is built with a modular, type-safe architecture emphasizing performance, reliability, and extensibility.

### Core Components

- **Server Layer**: Implements the MCP v1 specification with transport options (STDIO, HTTP/SSE)
- **Core Engine**: High-performance asynchronous crawling with memory-adaptive concurrency
- **CrawlerManager**: Central coordinator for all crawling operations with state management
- **HttpClientManager**: Optimized, connection-pooled HTTP client with retry logic and stats
- **Extraction Pipeline**: Processes raw web content into LLM-friendly formats
- **Resource Manager**: Handles MCP resources with persistent storage and retrieval

### Memory-Adaptive Concurrency

The system automatically adjusts concurrency based on available system resources:

```bash
# Example configuration with memory-adaptive settings
crawl4ai-mcp serve --max-memory-percent 75 --min-concurrent-requests 5 --max-concurrent-requests 25
```

### Modern Python Standards

- **Type Safety**: Comprehensive typing with runtime validation via Pydantic v2
- **Async-First**: Built on asyncio with proper concurrency patterns
- **Testing**: Extensive pytest suite with property-based testing
- **Documentation**: Google-style docstrings with complete type annotations
- **Dependency Management**: Modern pyproject.toml with optional-dependencies
