## 1. Introduction {#s-1-introduction}

### 1.1 Purpose {#s-11-purpose}
This document outlines the product requirements, features, technical specifications, and overall vision for **Crawl4AI MCP**, a server implementation designed to bridge the powerful web crawling and data extraction capabilities of the Crawl4AI library with the **Model Context Protocol (MCP)** standard. It serves as a central reference for development, testing, and stakeholders.

### 1.2 Product Overview {#s-12-product-overview}
Crawl4AI MCP functions as an **intelligent gateway**, enabling Large Language Models (LLMs) such as Claude to interface dynamically with the live web. By adhering to the MCP standard, it empowers AI models to utilize Crawl4AI's comprehensive engine for **retrieving, processing, extracting, and summarizing web content** directly within their operational context. This capability significantly enhances LLMs by grounding their outputs and reasoning in real-time, structured, and contextually relevant web data, thereby overcoming the limitations of static training datasets.

The system leverages Crawl4AI's powerful features, including:
- **Concurrency-based BFS crawling** with configurable parallelism for efficient site traversal
- **Advanced rate-limiting mechanisms** with customizable **and adaptive delays (including randomized jitter and exponential backoff)** to prevent IP blocking **and respect server load**
- **Comprehensive sitemap discovery and parsing** for complete site coverage
- **Multiple extraction strategies** (CSS, XPath, LLM-based, Cosine similarity)
- **Rich output formatting** with **structure-aware Markdown preserving semantic HTML elements**, JSON, HTML, and TXT options
- **Structured data extraction** using customizable schemas
- **Caching mechanisms** with multiple modes (ENABLED, DISABLED, READ_ONLY, WRITE_ONLY, BYPASS)
- **Browser automation** with support for Chromium, Firefox, and WebKit
- **File chunking and processing** optimized for RAG pipelines and LLM ingestion **(including semantic chunking based on sentence similarity and configurable overlap)**

### 1.3 Background {#s-13-background}
The contemporary AI landscape is characterized by rapid evolution. While Large Language Models (LLMs) possess extensive knowledge derived from their training data, their ability to access and utilize **current, dynamic web information** remains a significant challenge, often relying on indirect or outdated methods. This limitation hinders their applicability in scenarios requiring real-time data awareness.

The **Model Context Protocol (MCP)**, an open standard initiated by Anthropic, provides a crucial solution by defining a secure and standardized framework for AI models to interact predictably with external tools and data sources, including the live web.

Concurrently, **Crawl4AI** has emerged as a premier open-source Python library, meticulously engineered for web crawling and data extraction specifically tailored for AI applications. It offers sophisticated capabilities such as deep crawling, robust handling of dynamic web content (JavaScript rendering), and the generation of **structure-aware, LLM-optimized markdown outputs**.

This project, **Crawl4AI MCP**, strategically integrates these two pivotal technologies. Its objective is to deliver a state-of-the-art MCP server implementation optimized for web interaction. By acting as a vital bridge, it connects advanced LLMs with the vast, continuously evolving information landscape of the World Wide Web, addressing the escalating demand for AI systems capable of operating on, reasoning with, and extracting tangible value from live online data, **while adhering to stringent ethical and performance standards**.

### 1.5 Scope {#s-15-scope}

#### 1.5.1 In Scope {#s-151-in-scope}
- **MCP Protocol Implementation:** Full compliance with MCP v1 server specification for tools and resources (listing, calling, reading). Authentication via standard MCP mechanisms if applicable (e.g., headers, **potentially OAuth 2.1 if mandated by spec updates**).
- **Core Crawling Engine:** Integration with `crawl4ai` library as the backend for all web interaction tasks.
- **Transport Layers:** Support for STDIO, HTTP (with SSE for streaming), and WebSocket transports via `fastmcp`. **Monitor and adapt to potential future MCP standard transport changes (e.g., Streamable HTTP, JSON-RPC batching).**
- **MCP Tools:** Implementation of the following tools with configurable parameters:
    - `crawl_url`: Single page fetching, processing, **semantic chunking**, and optional structured extraction.
    - `deep_crawl`: Recursive crawling with depth/page limits, domain/pattern filtering, and strategy selection (BFS, DFS, BFF).
    - `extract_structured_data`: Targeted data extraction using CSS, XPath, or LLM-based schemas.
    - `get_page_metadata`: Fetching metadata (title, description, etc.) without full content processing.
    - `save_website`: Saving crawled content (single, sitemap, or limited crawl) to local files in various formats (**structure-aware MD**, JSON, HTML, TXT) with optional resource downloading.
    - `generate_knowledge_base`: Processing crawled content into **semantically chunked** files (MD, JSON) with metadata for RAG pipelines.
    - `batch_crawl`: Efficient processing of multiple URLs, potentially triggering other actions like saving or KB generation.
    - `search` *(Basic V1)*: Keyword search within results of the current session/batch.
- **MCP Resources:** Implementation of resource providers for:
    - `crawl://results/{session_id}`: Accessing summaries or full details of crawl results.
    - `crawl://schemas/{schema_name}`: Reading predefined extraction schema definitions.
    - `crawl://images/{image_id}`: Retrieving cached image data (if enabled).
- **Configuration:** Hierarchical configuration system (Defaults < YAML < Env Vars < CLI args) for server settings, crawler behavior, authentication, **ethical crawling parameters**, etc.
- **Deployment Options:**
    - `pip install crawl4ai-mcp` for package installation.
    - Docker container image with `Dockerfile` and example `docker-compose.yml`.
- **Documentation:** Comprehensive guides (Quick Start, Configuration, Tools API, Deployment, **Ethical Crawling Guide**) and practical examples.
- **Testing:** Robust test suite (Unit, Integration, E2E) ensuring functionality, reliability, and MCP compliance (>85% coverage target). **Include tests for ethical compliance features.**
- **Ethical Crawling:** Default adherence to `robots.txt`, **configurable and adaptive rate limiting (with jitter/backoff)**, **clear and configurable User-Agent identification**, **optional checking of ToS/meta tags**, **handling of specific HTTP codes (403, 429)**, **data minimization principles**.

#### 1.5.2 Out of Scope (Initially) {#s-152-out-of-scope-initially}
- Advanced user management and role-based access control (RBAC).
- Graphical user interface (GUI) for server management or configuration.
- Direct integration with specific databases or vector stores beyond file-based exports (though generated KBs can be ingested).
- Real-time collaborative crawling features.
- Advanced AI-driven crawling strategies beyond Crawl4AI's built-in capabilities (e.g., reinforcement learning for crawling).
- Built-in support for languages **other than Python** for the server implementation.
- Distributed crawling across multiple server instances (focus on single-instance reliability first).
- **Advanced Job Scheduling/Queueing:** While crawls can be triggered via MCP, the server itself won't include a complex cron-like scheduler or persistent job queue (external orchestration is expected).
- **Automated ToS/Privacy Policy Parsing & Interpretation:** Automatically understanding and enforcing rules from human-readable ToS is complex and legally risky, remaining out of scope. Configuration for respecting known constraints is in scope.

### 2.5 User Stories and Use Cases {#s-25-user-stories-and-use-cases}

<Accordions>
<Accordion title="2.5.1 Key User Stories {#s-251-key-user-stories}">
**US-01: Web Content Extraction for LLM Processing**
*As an* AI researcher,
*I want to* extract clean, relevant, **semantically chunked, and structure-aware** content from web pages,
*So that* I can feed high-quality data to my language models without extensive preprocessing **and maximize contextual understanding**.

**US-02: Structured Data Extraction**
*As an* AI application developer,
*I want to* extract specific structured data from websites according to defined schemas,
*So that* my AI agents can work with consistent, well-formatted information.

**US-03: Deep Website Crawling**
*As a* data analyst,
*I want to* automatically crawl entire websites or sections to specified depths,
*So that* I can collect comprehensive datasets without manual navigation.

**US-04: Knowledge Base Generation**
*As an* AI researcher,
*I want to* create searchable knowledge bases from multiple web sources, **chunked semantically and including relevant metadata**,
*So that* my models can access and reference relevant information efficiently **for RAG applications**.

**US-05: Website Content Monitoring**
*As a* data analyst,
*I want to* track changes on specific web pages over time,
*So that* I can identify and analyze updates to competitive information.

<Accordion title="2.5.2 Core Use Cases {#s-252-core-use-cases}">
**UC-01: Single URL Content Extraction**
1.  User submits a URL through an MCP client using the `crawl_url` tool
2.  MCP server validates the URL and crawling parameters
3.  Server retrieves the web content, respecting robots.txt and **configured ethical crawling practices (rate limits, User-Agent, ToS check flags)**
4.  Server processes the HTML, extracting main content, removing boilerplate, **generating structure-aware markdown, and applying semantic chunking**.
5.  Server returns clean, processed content (markdown or JSON) to the client
6.  MCP client displays or processes the content according to the user's needs

**UC-02: Structured Data Extraction**
1.  User defines a schema for specific data points (e.g., product information, pricing)
2.  User submits URL(s) and schema through an MCP client using the `extract_structured_data` tool
3.  Server navigates to the URL and analyzes the page structure
4.  Server extracts data according to the provided schema or selectors
5.  Server validates and formats the extracted data as structured JSON
6.  Server returns the structured data to the client for further processing

**UC-03: Deep Crawling and Knowledge Base Creation**
1.  User specifies a starting URL, crawl depth, and configuration parameters
2.  User initiates deep crawling via the `deep_crawl` or `generate_knowledge_base` tool
3.  Server begins crawling from the start URL, following links according to specified parameters
4.  Server provides streaming progress updates to the client
5.  Server processes and stores content from all crawled pages
6.  Server organizes the content into a **semantically chunked and structured** knowledge base format
7.  Server returns a comprehensive knowledge base or collection of processed pages

**UC-04: Real-time Web Research**
1.  User asks an AI assistant a question requiring current web information
2.  AI assistant (via MCP client) identifies relevant URLs to check
3.  Client requests content from these URLs using the `crawl_url` tool
4.  Server retrieves and processes the content from each URL **ethically**
5.  Server returns clean, **semantically relevant** content to the AI assistant
6.  AI assistant incorporates the web information into its response

**UC-05: Batch URL Processing**
1.  User provides a list of URLs to process with specific extraction parameters
2.  User initiates batch processing through the `batch_crawl` tool
3.  Server schedules and executes crawling of all URLs with appropriate **adaptive** rate limiting **and ethical checks**.
4.  Server processes each page according to extraction parameters, **applying semantic chunking**.
5.  Server provides progress updates as URLs are processed
6.  Server returns all processed results in a consistent format

**UC-06: Accessing Crawl Results as Resources**
1.  User performs a crawl operation that generates results
2.  Server stores the results and makes them available as MCP resources
3.  User or AI assistant accesses specific parts of results via the resource system
4.  Server provides the requested resource data according to MCP standards
5.  Client or AI assistant uses the resource data as needed
</Accordion>
</Accordions>

## 9. Constraints and Assumptions {#s-9-constraints-and-assumptions}

### 9.1 Constraints {#s-91-constraints}
- **Dependency on `crawl4ai`:** Core crawling logic relies heavily on the `crawl4ai` library. Its limitations or bugs may impact this project.
- **Dependency on `fastmcp`:** MCP protocol handling and transport layers depend on the `fastmcp` library.
- **External Website Variability:** Target websites can change structure, implement anti-bot measures, or become unavailable, impacting crawl success.
- **Resource Limits:** Performance is constrained by server hardware (CPU, RAM, Network I/O) and browser resource usage.
- **Ethical Boundaries:** Must adhere to ethical crawling practices (`robots.txt`, rate limiting, **respect for ToS/meta tags where configured**), potentially limiting crawl speed or scope. Compliance with data privacy regulations (GDPR, CCPA) is essential if handling PII.
- **MCP Standard Evolution:** Future changes to the MCP standard (authentication, transport, features) may require adaptation. **Dependency on `fastmcp` for implementing these changes.**

### 9.2 Assumptions {#s-92-assumptions}
- Users have a basic understanding of web crawling concepts and the MCP protocol.
- Users can provide necessary configurations (e.g., API keys for LLM tools if used).
- The `crawl4ai` and `fastmcp` libraries will be maintained and remain compatible.
- Standard web technologies (HTML, CSS, basic JS) are used by most target websites.
- Network connectivity is available for the server to reach target websites.
- Users will run the server in environments meeting the minimum Python version requirements.
- Users will configure the tool responsibly, especially regarding ethical crawling parameters (rate limits, User-Agent), respecting website terms and legal requirements.
